The Problem

Cold wallets are secure.
But blind.

  • Best physical security in crypto — keys never touch the internet
  • Zero counterparty intelligence — no idea who you're sending to
  • Once signed offline, transactions are irreversible
  • Every cold wallet today: Ledger, Trezor, Keystone — no reputation layer

Market Gap

$1.2B+ hardware wallet market

Zero products integrate on-chain reputation into the signing flow. Users are protected from key theft but fully exposed to social engineering, address poisoning, and scam recipients.

6 FairScore Integration Points

FairScore is not decorative. It is a core gating mechanism.

1. Transaction Gating

Bronze = hard block. Silver = soft warning. Checked before every outbound transfer at the air-gap boundary.

2. Dynamic Limits

Transfer caps scale with reputation tier. Bronze: 0 SOL. Silver: 10. Gold: 100. Platinum: 500. Diamond: unlimited.

3. DAO Governance

Multi-sig vote weight influenced by voter's FairScore. Tier 5 = 2x multiplier. Tier 1 = cannot vote. Anti-Sybil.

4. Jupiter Screening

Token deployer addresses checked before swap creation. Low-tier contracts trigger warnings against honeypots and rugs.

5. Dashboard Badges

Color-coded reputation indicators in the portfolio panel. Every token issuer shows trust level inline.

6. MCP Agent Gates

AI agent autonomy scales with FairScore. Tier 3+ can execute. Tier 1-2 blocked. Prevents compromised agent drain.

FairScore Flow

  • User enters recipient address
  • Online device queries FairScale API
  • Score embedded in QR payload
  • Offline device shows score for verification
  • User confirms → signs → returns via QR

Key Design Decisions

  • Air-gap device NEVER contacts API
  • 5-minute TTL response cache
  • Graceful degradation on API failure
  • Real FairScale API (api2.fairscale.xyz)
